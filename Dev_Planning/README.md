# Dev_Planning - Development Branch Only

## Purpose

This folder contains all planning and sprint-related files for the Dinner First project. These files are meant to exist **only in the development branch** and should never be merged into main.

## Contents

- **SPRINT_PLAN.md** - Main sprint planning document for critical feature implementation
- **sprint2_aug20th_2025.md** - Sprint 2 comprehensive test coverage implementation
- **sprint-3-2025.md** - Sprint 3 integration, testing & production deployment
- **roadmap2025.md** - Complete production roadmap for 2025
- **Sprint6_2025.md** - Sprint 6 test infrastructure enhancement & coverage expansion

## Branch Strategy

### Development Branch
- Contains all planning files
- Used for sprint planning and tracking
- Internal team documentation
- Implementation roadmaps and timelines

### Main Branch
- Production-ready code only
- No planning or internal documentation
- Clean, focused on actual implementation
- Public-facing repository structure

## Important Note

When creating PRs from development to main, ensure that the `Dev_Planning/` folder is **excluded** from the merge. This keeps main branch clean and focused on code implementation.

## Usage

1. All sprint planning happens in this folder
2. Track progress using these documents
3. Update sprint files as work progresses
4. Archive completed sprints for historical reference
